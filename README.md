# 🔄 Tableau → Power BI Migration

### From Excel MIS → Tableau → Power BI | A Real BFSI Migration Story

This project demonstrates how an enterprise Wealth Management MIS was migrated from **manual Excel reporting** to **Tableau**, and then completely rebuilt in **Power BI** while preserving the same business logic, KPIs, and executive reporting standards.

Instead of redesigning reports from scratch, the objective was to prove that a governed BI solution should remain consistent regardless of the BI platform.

## 🎯 Project Objective

A Wealth Banking team managing **₹21.54B Assets Under Management (AUM)** across **100 Wealth Managers** and **53 Branches** relied entirely on manual Excel MIS reports.

The migration demonstrates how the same analytical solution can be implemented across multiple BI platforms without changing the underlying business logic.

---

## ❗ Business Problem

The existing reporting process had several limitations:

- Reports required nearly **3 days** to prepare.
- No real-time visibility into zone performance.
- No early identification of underperforming Wealth Managers.
- Leadership decisions relied on disconnected Excel reports.
- Product performance and risk analysis were unavailable.

This was not simply a reporting issue—it was a business decision-making challenge.

---

## 🚀 Migration Journey

### Phase 1 — Excel → Tableau

- Automated Wealth MIS reporting
- Executive KPI Dashboard
- Zone Performance Analysis
- Product Mix Analysis
- LOD Expressions
- Interactive Executive Dashboards

### Phase 2 — Tableau → Power BI

The complete Tableau solution was rebuilt in Power BI using:

- Power Query
- Star Schema Data Model
- DAX Measures
- Time Intelligence
- Row-Level Security (RLS)
- Interactive Cross Filtering
- Drill-through Navigation

The objective was to demonstrate that business logic remains portable across BI platforms.

> **Business Intelligence is not about dashboards. It is about governed business logic.**
